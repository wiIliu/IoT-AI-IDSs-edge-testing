Workflow

  1. Data: RT‑IoT‑2022 (benign vs. malicious). Include DOI/citation in the repo.
  2. Preprocessing: cleaning, feature engineering/selection, scaling where useful.
  3. Splits: train/validation/test (e.g., 70/15/15).
  4. Models:
    • ML: Decision Tree, Random Forest, XGBoost
    • DL: RNN/LSTM to capture temporal dependencies
  5. Evaluation: Accuracy, Precision, Recall, F1, ROC‑AUC; confusion matrices; PR/ROC curves.

Feature Pipeline

Document feature order and any normalization so deployment matches training. Consider exporting a preproc.json with mean/std and exact feature order.

Training Notes

  • Hyperparameter search (grid/random/Bayes)
  • Early stopping and class‑weighted loss (if needed)
  • Seeded runs for reproducibility

Baselines & Checks

  • Majority baseline, simple tree baseline
  • Leakage and data drift checks
  • k‑fold where appropriate
